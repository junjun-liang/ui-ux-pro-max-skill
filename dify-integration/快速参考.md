# UI/UX Pro Max - Dify集成快速参考

## 🚀 快速开始

### 一键部署（Docker）

```bash
# 1. 进入集成目录
cd dify-integration

# 2. 运行部署脚本
./deploy-to-dify.sh docker

# 3. 在Dify中添加工具
#    - 打开Dify管理后台
#    - 创建/编辑应用
#    - 添加"UI/UX Pro Max"工具
```

### 本地开发部署

```bash
# 1. 进入集成目录
cd dify-integration

# 2. 运行部署脚本
./deploy-to-dify.sh local

# 3. 启动Dify API
cd ../dify/api
python main.py
```

## 🛠️ 可用工具

### 1. generate_design_system

**功能**: 生成完整的设计系统推荐

**参数**:
- `query` (必需): 搜索查询
- `project_name` (可选): 项目名称
- `output_format` (可选): "markdown" 或 "json"

**示例**:
```json
{
  "query": "美容SPA wellness",
  "project_name": "Serenity Spa",
  "output_format": "markdown"
}
```

**输出**:
```markdown
## Design System: Serenity Spa

### Pattern
- **Name:** Hero-Centric + Social Proof
- **CTA Placement:** Above fold
- **Sections:** Hero > Features > CTA

### Style
- **Name:** Soft UI Evolution
- **Keywords:** Evolved soft UI, better contrast, modern aesthetics
- **Best For:** Modern enterprise apps, SaaS platforms, health/wellness

### Colors
| Role | Hex |
|------|-----|
| Primary | #2563EB |
| Secondary | #3B82F6 |
| CTA | #F97316 |
| Background | #F8FAFC |
| Text | #1E293B |

### Typography
- **Heading:** Lora
- **Body:** Raleway
- **Mood:** calm, wellness, health, relaxing, natural, organic

### Key Effects
Improved shadows (softer than flat, clearer than neumorphism), modern (200-300ms), focus visible, WCAG AA/AAA

### Avoid (Anti-patterns)
- Bright neon colors
- Harsh animations
- Dark mode

### Pre-Delivery Checklist
- [ ] No emojis as icons (use SVG: Heroicons/Lucide)
- [ ] cursor-pointer on all clickable elements
- [ ] Hover states with smooth transitions (150-300ms)
- [ ] Light mode: text contrast 4.5:1 minimum
- [ ] Focus states visible for keyboard nav
- [ ] prefers-reduced-motion respected
- [ ] Responsive: 375px, 768px, 1024px, 1440px
```

### 2. search_domain

**功能**: 搜索特定UI/UX域

**参数**:
- `query` (必需): 搜索查询
- `domain` (可选): 搜索域（自动检测）
- `max_results` (可选): 最大结果数（默认3）
- `output_format` (可选): "markdown" 或 "json"

**可用域**:
- `style`: UI风格
- `color`: 配色方案
- `typography`: 字体配对
- `chart`: 图表类型
- `landing`: 落地页模式
- `product`: 产品类型
- `ux`: UX指南
- `google-fonts`: Google字体
- `icons`: 图标库
- `react`: React性能
- `web`: 应用界面指南

**示例**:
```json
{
  "query": "glassmorphism",
  "domain": "style",
  "max_results": 3,
  "output_format": "markdown"
}
```

### 3. search_stack_guidelines

**功能**: 搜索技术栈特定指南

**参数**:
- `query` (必需): 搜索查询
- `stack` (必需): 技术栈
- `max_results` (可选): 最大结果数（默认3）
- `output_format` (可选): "markdown" 或 "json"

**可用技术栈**:
- `html-tailwind`: HTML + Tailwind CSS
- `react`: React
- `nextjs`: Next.js
- `vue`: Vue
- `nuxtjs`: Nuxt.js
- `nuxt-ui`: Nuxt UI
- `svelte`: Svelte
- `astro`: Astro
- `swiftui`: SwiftUI
- `react-native`: React Native
- `flutter`: Flutter
- `shadcn`: shadcn/ui
- `jetpack-compose`: Jetpack Compose

**示例**:
```json
{
  "query": "form validation",
  "stack": "react-native",
  "max_results": 3,
  "output_format": "markdown"
}
```

## 🎯 使用场景

### 场景1: 生成设计系统 + HTML代码

**用户输入**: "帮我做一个美容SPA网站的首页"

**工作流**:
1. LLM分析需求 → 提取产品类型、风格偏好
2. 调用`generate_design_system` → 获取设计系统
3. LLM根据设计系统生成HTML代码
4. 输出完整HTML

**Dify工作流配置**:
```yaml
nodes:
  - id: "analyze"
    type: "llm"
    prompt: "分析用户需求，提取产品类型和风格偏好"
  
  - id: "generate_design_system"
    type: "tool"
    tool: "generate_design_system"
    parameters:
      query: "{{analyze.product_type}} {{analyze.style_preference}}"
      project_name: "{{analyze.project_name}}"
  
  - id: "generate_html"
    type: "llm"
    prompt: "根据设计系统生成HTML代码"
    input: "{{generate_design_system.output}}"
```

### 场景2: 搜索UI风格推荐

**用户输入**: "推荐一些适合SaaS产品的UI风格"

**工作流**:
1. LLM识别产品类型（SaaS）
2. 调用`search_domain` → 搜索风格
3. LLM总结推荐
4. 输出风格列表

**Dify工作流配置**:
```yaml
nodes:
  - id: "identify"
    type: "llm"
    prompt: "识别产品类型和风格偏好"
  
  - id: "search_styles"
    type: "tool"
    tool: "search_domain"
    parameters:
      query: "{{identify.product_type}} modern professional"
      domain: "style"
      max_results: 5
  
  - id: "summarize"
    type: "llm"
    prompt: "总结风格推荐"
    input: "{{search_styles.output}}"
```

### 场景3: 技术栈最佳实践

**用户输入**: "React Native表单验证的最佳实践是什么？"

**工作流**:
1. LLM识别技术栈（React Native）和主题（表单验证）
2. 调用`search_stack_guidelines` → 搜索指南
3. LLM总结最佳实践
4. 输出指南列表

**Dify工作流配置**:
```yaml
nodes:
  - id: "identify"
    type: "llm"
    prompt: "识别技术栈和主题"
  
  - id: "search_guidelines"
    type: "tool"
    tool: "search_stack_guidelines"
    parameters:
      query: "{{identify.topic}}"
      stack: "{{identify.stack}}"
      max_results: 3
  
  - id: "summarize"
    type: "llm"
    prompt: "总结最佳实践"
    input: "{{search_guidelines.output}}"
```

## 🔧 配置示例

### Dify工作流完整配置

```yaml
app:
  name: "UI/UX Design Assistant"
  description: "AI-powered UI/UX design assistant with UI/UX Pro Max"

workflow:
  nodes:
    # 开始节点
    - id: "start"
      type: "start"
      data:
        title: "开始"
        desc: "用户输入设计需求"

    # 需求分析
    - id: "analyze_requirements"
      type: "llm"
      model:
        provider: "openai"
        name: "gpt-4"
      prompt: |
        分析用户的设计需求，提取以下信息：
        1. 产品类型（如：SaaS、电商、美容SPA等）
        2. 风格偏好（如：现代、优雅、极简等）
        3. 技术栈（如：HTML+Tailwind、React、Vue等）
        4. 项目名称（可选）
        
        用户需求: {{#start.text#}}
        
        请以JSON格式输出：
        {
          "product_type": "...",
          "style_preference": "...",
          "tech_stack": "...",
          "project_name": "..."
        }

    # 生成设计系统
    - id: "generate_design_system"
      type: "tool"
      provider: "ui_ux_pro_max"
      tool: "generate_design_system"
      parameters:
        query: "{{#analyze_requirements.output.product_type#}} {{#analyze_requirements.output.style_preference#}}"
        project_name: "{{#analyze_requirements.output.project_name#}}"
        output_format: "markdown"

    # 搜索配色方案
    - id: "search_colors"
      type: "tool"
      provider: "ui_ux_pro_max"
      tool: "search_domain"
      parameters:
        query: "{{#analyze_requirements.output.product_type#}} {{#analyze_requirements.output.style_preference#}}"
        domain: "color"
        max_results: 2
        output_format: "markdown"

    # 搜索字体配对
    - id: "search_typography"
      type: "tool"
      provider: "ui_ux_pro_max"
      tool: "search_domain"
      parameters:
        query: "{{#analyze_requirements.output.product_type#}} {{#analyze_requirements.output.style_preference#}}"
        domain: "typography"
        max_results: 2
        output_format: "markdown"

    # 搜索技术栈指南
    - id: "search_stack_guidelines"
      type: "tool"
      provider: "ui_ux_pro_max"
      tool: "search_stack_guidelines"
      parameters:
        query: "responsive layout accessibility"
        stack: "{{#analyze_requirements.output.tech_stack#}}"
        max_results: 3
        output_format: "markdown"

    # 生成HTML代码
    - id: "generate_code"
      type: "llm"
      model:
        provider: "openai"
        name: "gpt-4"
      prompt: |
        你是一个专业的前端开发工程师。根据以下设计系统生成HTML代码。
        
        ## 设计系统
        {{#generate_design_system.output#}}
        
        ## 配色方案
        {{#search_colors.output#}}
        
        ## 字体配对
        {{#search_typography.output#}}
        
        ## 技术栈指南
        {{#search_stack_guidelines.output#}}
        
        请生成完整的HTML代码，包括：
        1. HTML结构
        2. CSS样式（使用Tailwind CSS）
        3. 响应式设计
        4. 无障碍支持
        5. 性能优化
        
        技术栈: {{#analyze_requirements.output.tech_stack#}}

    # 代码审查
    - id: "review_code"
      type: "llm"
      model:
        provider: "openai"
        name: "gpt-4"
      prompt: |
        审查以下HTML代码，检查是否符合设计系统要求：
        
        ## 生成的代码
        {{#generate_code.output#}}
        
        ## 检查清单
        - [ ] No emojis as icons (use SVG: Heroicons/Lucide)
        - [ ] cursor-pointer on all clickable elements
        - [ ] Hover states with smooth transitions (150-300ms)
        - [ ] Light mode: text contrast 4.5:1 minimum
        - [ ] Focus states visible for keyboard nav
        - [ ] prefers-reduced-motion respected
        - [ ] Responsive: 375px, 768px, 1024px, 1440px
        
        请输出检查结果和改进建议。

    # 结束节点
    - id: "end"
      type: "end"
      outputs:
        - title: "HTML代码"
          value: "{{#generate_code.output#}}"
        - title: "代码审查"
          value: "{{#review_code.output#}}"

  edges:
    - source: "start"
      target: "analyze_requirements"
    - source: "analyze_requirements"
      target: "generate_design_system"
    - source: "analyze_requirements"
      target: "search_colors"
    - source: "analyze_requirements"
      target: "search_typography"
    - source: "analyze_requirements"
      target: "search_stack_guidelines"
    - source: "generate_design_system"
      target: "generate_code"
    - source: "search_colors"
      target: "generate_code"
    - source: "search_typography"
      target: "generate_code"
    - source: "search_stack_guidelines"
      target: "generate_code"
    - source: "generate_code"
      target: "review_code"
    - source: "review_code"
      target: "end"
```

## 🐛 故障排除

### 问题1: 工具未显示

**症状**: 在Dify工具列表中看不到UI/UX Pro Max

**解决**:
```bash
# 检查文件是否存在
docker exec dify-api-1 ls -la /app/api/core/tools/provider/builtin/ui_ux_pro_max/

# 重启API服务
docker restart dify-api-1

# 清除缓存
docker exec dify-api-1 rm -rf /app/api/core/tools/provider/builtin/__pycache__
```

### 问题2: CSV文件未找到

**症状**: 工具报错"File not found: products.csv"

**解决**:
```bash
# 检查数据目录
docker exec dify-api-1 ls -la /app/api/core/tools/provider/builtin/ui_ux_pro_max/data/

# 重新复制数据文件
cd dify-integration
./deploy-to-dify.sh docker
```

### 问题3: Python导入错误

**症状**: 工具调用失败，显示导入错误

**解决**:
```bash
# 测试Python脚本
docker exec dify-api-1 python3 -c "import sys; sys.path.insert(0, '/app/api/core/tools/provider/builtin/ui_ux_pro_max'); import ui_ux_pro_max; print('OK')"

# 检查Python版本
docker exec dify-api-1 python3 --version

# 检查依赖
docker exec dify-api-1 pip list | grep -E "csv|json|pathlib"
```
